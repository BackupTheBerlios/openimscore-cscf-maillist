Hi Naveen,

> When I register, say with a URI 'sip:alice at open-ims.test', where is this
> registration information stored?

That information is distributed in IMS architecture compared to plain SIP,
where the registrar (location database) stores all information about the
reachability of a user.

In IMS the HSS stores information which S-CSCF is assigned to a subscriber
based on its private user identity and which public user identity is
actually registered.
The assigned S-CSCF stores information on how a public user identity is
reachable in terms of CSCF-s involved (Path header).
The assigned P-CSCF stores information on how the user equipment of a public
user identity is reachable in terms of IP-transport (ports, security
association).

> I browsed through the HSS Database but could not find any entry with
> current location of alice with which alice has registered.

The storage mechanism (persistent in a database or not) of above mentioned
"registration information" is a different question (implementation
dependent).
