Well, yes, there is, by changing the scscf.cfg. But then, you might run
into some issues on the initial registration... not sure about it
though, never tried this myself, but I received some feedback that
indicated a lot of open issues.

For subsequent registration, if you have a client that can do IPSec (or
if you hack the pcscf.cfg) then the Authorization header will contain a
"integrity-protected=yes" parameter. If the S-CSCF finds this, it will
not challenge you anymore as the channels that the request was received
is considered to be secure.
