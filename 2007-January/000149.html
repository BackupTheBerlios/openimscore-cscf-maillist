The UCT IMS Client expects to receive a 183 provisional response from an 
INVITE request (preconditions are a requirement), and not a 200OK.

Only once the IMS Client is happy that the QoS is properly setup will it 
send a 200OK. Take a close look at the signalling between two IMS 
clients. They can both run on the same machine, but will need to be 
registered with different user names.

Try running in SIP mode if you simply want INVITE -> OK -> ACK.
