Hi Andrea,

I think that you are taking the specs to strict. The 2 standards do not
conflict actually. So each identity can be used  by multiple UEs such
that you could have multiple contacts on the same identity.

Then the second one refers to a different thing - that the current specs
only support 1 IP address/UE (does not actually say 1 IP
address/identity!!!).

An UE as a concept is a different thing. I see it as one logical entity
having possibly multiple identities and this version supports only UEs
that use just 1 IP at once. That is not to say that you can not have
multiple logical UEs grouped in your IMS client, rather that each can
not just jump from one IP to other as it chooses. I would nterpret it
more like: you can not start some signaling from one IP and then expect
to continue it seamlessly from another...

Cheers,
-Dragos
