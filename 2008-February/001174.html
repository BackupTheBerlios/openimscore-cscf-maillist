Hi harishbansal1979,

> ....... If Bob is busy when someone calls him, AS server
> changes the response status code from 486 (Busy) to 302 (Temporary
> Unavailable) and specifies the sip uri of another subscriber.
> 
> Response status is successfully changed to 302 but SCSCF does not try
> to send the request to new contact.

This is not a bug. This is what is expected according to the standard.
Look at TS 23.228 §5.11.5, in particular look at the case §5.11.5.2a
(behaviour of S-CSCF#1).

If you want the redirect to be executed within the network, then you have to
use an AS. See TS 23.228 §5.11.5.5 as an example.
