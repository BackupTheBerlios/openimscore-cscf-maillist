Hi.

Check the output in FHoSS console. You should see something like this:


        2008-03-10 13:06:57,695 INFO  de.fhg.fokus.hss.main.TomcatServer - startTomcat Tomcat-Server is started.
        2008-03-10 13:06:59,056 WARN  org.apache.catalina.connector.MapperListener - registerEngine Unknown default host: 192.168.11.4
        2008-03-10 13:07:00,251 INFO  de.fhg.fokus.hss.web.servlet.ResponseFilter - init Response Filter Initialisation!
        2008-03-10 13:07:01,275 INFO  de.fhg.fokus.hss.main.TomcatServer - startTomcat WebConsole of FHoSS was started !


The IP in the second line showed, must be in keeping with hss.properties
host configuration. At the beginning of this file I have:


        # host & port : specify the IP address and the port where Tomcat is listening, e.g. host=127.0.0.1; port=8080;
        
        host=192.168.11.4
        port=8080


If you want to use localhost interface, you should see 127.0.0.1 in both
places (FHoSS output and hss.properties configuration file). Of course
it shouldn't be any error in FHoSS console. Also , with this kind of
interface you will only can work from the same machine that host the
Tomcat server embedded in FHoSS.

There is another question. If you have installed the FHoSS in Windows
box (possible although not very usual), you should check if Windows
firewall is in the middle.

Regards...
