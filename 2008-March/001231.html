searching the lists would have really helped you faster... my response

mustafa at cec.sy wrote:
> Hi all,
> i use a sip client (SJphone) in windows mobile 6 to register in Open IMS
> Core through GPRS connection but i have this error :
>
> ERR:P-CSCF:cscf_get_authorization: Message does not contain Authorization
> header.
>  3(7796) INF:P-CSCF:P_add_integrity_protected: No authorization header found.
>   
this is not necessary an error, but just an info. A 3GPP IMS client 
should be able to relay the IMPI information and without an 
Authorization header it is not possible. Search the mailing list for how 
is the IMPI derived from the IMPU in this case.
>  3(7796) ERROR: sip_hostport2su: could not resolve hostname: "open-ims.test"
>  3(7796) ERROR: uri2dst: failed to resolve "open-ims.test"
>  3(7796) ERROR: t_forward_nonack: failure to add branches
>   
This is self explanatory - your DNS does not work probably and it can 
not resolve the hostnames. Check it with dig for example.
