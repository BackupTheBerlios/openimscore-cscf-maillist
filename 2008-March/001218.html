Hi all,

 

I want know if could be possible to include one extra field into the
Route header by conf.

 

For example having this Route header:

 

Route:
<sip:172.24.6.172:5060;lr>,<sip:iscmark at scscf.open-ims.test.com:6060;lr;
s=1;h=0;d=0;a=7369703a616c6963654065732e696e>

 

I need add a "call" extra field like this, before send the message to
the Application Server:

 

Route:
<sip:172.24.6.172:5060;lr>,<sip:iscmark at scscf.open-ims.test.com:6060;lr;
s=1;h=0;d=0;a=7369703a616c6963654065732e696e>;call=orig

 

By the way I need to include an "orig" value for an originating call and
a "term" value for a terminating one into this new field.

 

Any Idea about if is that possible or if could be a new feature?

 

Thanks in advance for your support.

 

Victor.
