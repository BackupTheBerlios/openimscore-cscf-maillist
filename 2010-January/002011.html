Well, the clear text should be that sessions should not be left locked 
when sending messages. I will do the necessary changes in the I/S-CSCF...

Would be great if we'd have a manual, but you see how fast the API 
changes as we realize that it does not do what it should...

Keeping counts would be a bit overcomplicated. The we'll get into 
live-locks and probably it would be also harder to manage the code as 
you add the complexity of splitting the readers and writers...

Cheers,
-Dragos
