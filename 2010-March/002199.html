Hello

I have just started to play with openIMS and it seems to work fine
with Linksys phones (with soft clients like twinkle too),

with SNOM phone I have encountered following issue

-at the end of phone registration procedure S-CSCF is sending a NOTIFY
message to P-CSCF with reg event containing a XML info about
registered UA
-SNOM phone includes +sip.instance field in its contact header....
this field looks for example like this:

+sip.instance=\"<urn:uuid:564877cc-8a76-42c9-8907-43ac956b8da3>\"

-it is correctly sent in NOTIFY in <unknown_param name
="+sip.instance">.... but since it contains < and > characters it
seems to confuse XML parser on P-CSCF side causing:

500 Error encountered while processing notification

I think it should be possible to deal with it in
r_get_reginfo_full(...) in registrar_notify.c but Im too lame c++
programmer to say how :-D

Regards
Jan
