Hello Srini,


Actually, you have replied your first question with the second one. In order to be
sure that the all components work properly, watch out the log messages which
appear on your shells. There should be no errors. Then you have to think 
couple of scenarios to test your own IMS.Try registration of a user and call between 
registered users. If these work properly then you have successfully set up a
default testbed. Then you can try out things with reconfiguring your *.cfg files
for your special needs.

I dont use 3CX SIP Phone so i cannot exactly  tell you where your problem lies but
it's mostly because of a incorrect sip header which your client sends.
OpenIMSCore is built on IMS Standards so see if you can get an IMS Client for your
tests. This would ease your problems. Further more X-Lite Sip Phone has been also discussed 
in our forums again and again. So if you want to use a SIP-Phone, you would 
find enough guidance(but we dont give any support for that) in our mail archieves.

And for future, if you come up with any problems, please attach a trace of your
traffic so that we could help you with your problem.


I hope this helps,
Umut

ps: You dont have to cross-post the same mail in all forums, one is just enough ;)
