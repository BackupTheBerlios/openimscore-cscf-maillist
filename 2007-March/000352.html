Hi Ramachandran,

The actual implementation is asynchronous and we are synchronizing it
later. the reason was that for development it is a lot easier this way.
yes, this does influence the performance negatively, but:
1. the current performance levels for registration are not that bad
2. the target of the project is not a carrier grade deployment.
