Hi all,

 

I am using X-Lite 3.0 to talk to the Open IMS Core.

 

As I understand, MD5 works by the client sending a register first
without an Authorization header 

Then the server replies 401 Unauthorized first, followed by the client
sending the SIP REGISTER with the MD5 Authorization header

 

The PCSCF forwards registration requests to the SCSCF, but first it
checks if there is an Authorization header, and if not then it fails.
The SCSCF can be configured to prefer MD5 only, but the REGISTER never
gets to the SCSCF.

 

The PCSCF repeats this error:

 

1(16523) ERR:P-CSCF:cscf_get_authorization: Message does not contain
Authorization header.

 1(16523) INF:P-CSCF:P_add_integrity_protected: No authorization header
found.

 

How is this supposed to work?

 

P.S, not sure if it is related, I am not using IpSec

 

Thanks and best regards

Leigh
