Hi Nagaraj,

please monitor for Diameter, additionally from 3868, also port 3869 and
3870 if you use the default configuration files. By default,
wireshark/ethereal only dissects Diameter traffic on 3868, so this might
be why you missed it.

For the 2nd question: the I-CSCF is mostly state-less beyond SIP
transactions. Then the registration is not complete so you don't have
actually an S-CSCF attached after the MAA, just an authentication
pending flag. So you do need the 2nd UAR/UAA. The 2nd UAA might be
different as an S-CSCF name should've been saved for the pending
authentication and as such the I-CSCF would not do any selection (like
capabilities selection) for choosing a S-CSCF.

-Dragos
