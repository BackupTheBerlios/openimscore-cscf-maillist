Hi ,

When I am using UCT client on both ends, i am successful in registering user as well as making call.

But when I am using xlite at one end and UCT client at other I am facing problem with xlite getting registered. 
It is giving error "Registration error: 403- Forbidden HSS returned no authentication vector"

On analysis, I found that xlite and UCT client use different encryption algorithms. xlite uses "MD5" whereas UCT uses
"AKAv1-MD5". In scscf.cfg their is a clause for setting up "registration_default_algorithm" in which modparams can be 
set to either "AKAv1-MD5", "AKAv2-MD5" or "MD5".

So if I want to use xlite and UCT client, what should i need to do to make them talk to each other?


If any one faced such problem please  let me know.
