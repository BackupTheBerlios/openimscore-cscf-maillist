> SIP/2.0 400 Bad Request - P-Visited-Network-ID header missing
> Could any one tell me where it went wrong and how i should be debugging
it.

I guess that your client sends the request directly to the I-CSCF because of
DNS data, but it should send it to the P-CSCF.
You have to provide the target where to send the requests (P-CSCF) manually.
Don't let DNS decide because this leads to I-CSCF where requests are sent
to.
