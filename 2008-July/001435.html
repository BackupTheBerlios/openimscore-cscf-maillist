Hi everyone,

I've made some test between the Xlite and UCT IMS Client. The result are
quite positive but still, there are some parts that not work correctly.

First of all, I wanted to know if UCT IMS client handles MD5 authentication?
( I think YES)

Audio Call
If I make a call from Xlite to UCT Client, both phones are ringing but it
fails when I answer(in the UCT client part). The Xlite displays "Call
failed. Media negociation failed". So I directly went to the audio/video
codec menu to check whether they were the same on both side. But That did
not change anything. Attached is the wireshark trace (xlite2uctimscall).

If I make a call from UCT Client to Xlite, both phone are ringing and the
communication is established. But the only thing we can hear on both side is
a big noise! (Exactly the same when I used the UCTB2BUA...). Attached is the
wireshark trace(uctims2xlitecall).

Instant Messaging
If I write a message from UCT client to Xlite, everything works
well.Attached is the wireshark trace(xlite2uctimsIM).
But that is not working from Xlite to UCT Client. Xlite receives a "REQUEST
TIME OUT".I do not why the UCTIMS client does not respond with a "200 OK"
when it receives the a  "MESSAGE".Attached is the wireshark
trace(uctims2xliteIM).


That would be great if anyone of you could explain thoses issues (maybe
bugs...)
