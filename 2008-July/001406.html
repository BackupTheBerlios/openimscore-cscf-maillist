well, in theory, you could have a transaction stateless AS, which does 
not add a Via when it forwards the INVITE back to the S-CSCF. Then you 
should not see any responses...

However, I would strongly recommend that you don't do this. Haven't seen 
a stateless SIP proxy for a long time now as it's no longer such a big 
performance killer to be stateful... especially if you are an AS.
