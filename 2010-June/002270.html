Hello,

Many user agents, such as Ekiga, only include an "Authorization" header
if prompted by the server using the WWW-Authenticate header.

However, if the private and public identity are different, OpenIMSCore
fails to recognize the public identity as a valid user account and
replies with "403 Forbidden - HSS User Unknown" instead of "401
Unauthorized - Challenging the UE", and so the client gives up before
sending its Authorization header.

If the same private and public identity are used, it works.

If a client is used that includes the Authorization header from the
beginning, without being prompted, it works too (even if public and
private identity are different), but as far as I understood, the
authorization dialog should behave the same way as the one between web
browsers and web servers, where the client only sends the info if
requested by server.

Thanks,

Alain
