> Hello Sudhakar,

> The xml configuration files are for the
> CDiameterPeer and
> JavaDiameterPeer libraries. Now if you run the whole
> Open IMS Core on
> the same machine, then you can leave it as it was.
> If you want to use
> the diameter peers between different machines (eg.
> FHoSS accessible for
> Sh from an AS on another machine, on a CSCF
> somewhere else) then you
> need to change the respective xml.
> 
> Now by default, the Accept tag in the xml has a bind
> parameter to the
> localhost. If you drop this parameter it will just
> listen for
> connections on all interfaces. So dropping this
> parameter is a quick way
> of making the diameter peers listen everywhere. I
> did bind it by default
> to 127.0.0.1 as a security measure - for
> inexperienced users, I didn't
> want to open up their diameter peer to everyone.
> 
> -Dragos
