Hi all,

As part of the persistency support I had to save the dlg_t data also.
This data structure is quite nasty as it contains a lot of other
different structures. As such, I have preferred not to re-implement the
whole serialization done in the dialog module. However, those functions
were using the lib/cds functionality. So if you want to use this branch,
you will need to do the following steps:

cd /opt/OpenIMSCore/ser_ims/lib
make -f Makefile.ser install.

Then use the new {p|s}cscf.cfg and ALSO the new {p|s}cscf.sh!

This is required ONLY in the 01_persistency branch and only from
revision 123 on.

Now, who volunteers to write some code? I received quite many
participation requests, so if you have some free time, we have some
immediate tasks for you, like:

- migrate the whole dlg_state to dlg_t structures
- send B2B BYEs to all INVITE dialogs on de-registration
- go through the code and make modifications for tel URI support
