Dear list members,
dear developers,
I discovered a bug in the code/grammar parsing the (p)cscf config file. 
If there is a part of the FQDN starting with a number, the program 
crashes with a core-dump:
========== console output begin ==========
root at pcscf:/opt/OpenIMSCore# ./pcscf.sh
./pcscf.sh: line 4: setkey: command not found
./pcscf.sh: line 5: setkey: command not found
 0(11675) parse error (26,17-18): syntax error
 0(11675) parse error (26,17-18): invalid hostname
./pcscf.sh: line 7: 11675 Segmentation fault      (core dumped) 
LD_LIBRARY_PATH="/usr/local/lib/ser" /opt/OpenIMSCore/ser_ims/ser -f 
/opt/OpenIMSCore/pcscf.cfg -D -D
========== console output end ==========
this is line 26:
alias=pcscf.ims.3abc.org:5060

Thus, the error points exactly to the "3" in the name and indeed, if I 
remove the "3" from the name (and reconfigure everything else 
accordingly), it works!

As we would need this for a domain which is already in place, two 
questions suggest itself:
1) [towards the developers ;-)) ] will this be fixed and if yes, how 
long will this take (worst case assumption), and
2) [towards everybody] can I work around this somehow? Like rewriting 
"From: ", "To: ", ... headers before forwarding or the like?

Thanks in advance & best regards,
Hannes.
