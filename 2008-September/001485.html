Hi Franz,
I disagree. The Text you cited is taken from the paragraph labeled 
"Preferred name syntax". Consequently I think that this "must" must be 
interpreted in the context of the paragraph, which is only a 
recommendation. I am sure the definition of the DNS namespace is much 
more complex than just the 6 lines given in RFC-1035.
In contrast to the text cited, the same paragraph reads:

The DNS specifications attempt to be as general as possible in the rules
for constructing domain names.  The idea is that the name of any
existing object can be expressed as a domain name with minimal changes.


... which means quite the opposite.

But the most important fact that your interpretation cannot be correct 
is that in this case you could NEVER do a DNS reverse lookup!

Moreover there are a lot of 2nd level domains starting with a number, 
like "6net.org" or "3gpp.org" and I doubt that Root Server 
administrators disobey the rules of the DNS namespace.

Therefore OSIMS must accept DNS names where one or more parts of it 
start with a number. Apart from that, core-dumping is always a bug in 
the software gegardless of its intended behavior.

Best regards,
Hannes.
