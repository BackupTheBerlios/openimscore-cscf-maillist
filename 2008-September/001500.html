> Then, I tried to change the domain to another one, to test roaming calls.
> I used the configurator.sh to change my cfg, XML and sh files. The new
> domain is sigrist.com.

Did you also change the sql-script files?

I always apply the configurator.sh script to the following three
directories:

- /opt/OpenIMSCore/ser_ims/cfg
  changing: icscf.cfg icscf_pg.sql icscf.sql icscf.thig.cfg icscf.xml
pcscf.cfg pcscf.xml persist_my.sql persist_pg.sql scscf.cfg scscf.xml
- /opt/OpenIMSCore/FHoSS/deploy
  changing: c3p0.properties DiameterPeerHSS.xml hibernate.properties
hss.properties log4j.properties
- /opt/OpenIMSCore/FHoSS/scripts
  changing: hss_db.sql userdata.sql

You must also re-install the database.

Be aware that the configurator.sh script is designed to be used only once.
If you apply it a second time you have to change some variables inside of
the script which are pre-configured for openims.test domain. But as you
perhaps have inconsistent files (some are already modified, some not) you
should best do the installation again, or check all the files mentioned
above manually.
