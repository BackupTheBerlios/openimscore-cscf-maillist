> For re-invite Do I have to add the two route header to send back to AS

For re-INVITE (a request within the dialog) you always have to use the
dialog route. This is the route setup during the dialog initiating
request
(the first INVITE) and is based on the Record-Route header fields
included
by the network nodes traversed. 

If the AS "record-routed" then it has to be included, otherwise not.
