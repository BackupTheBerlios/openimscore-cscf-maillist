Hi,

 

I am facing a problem with DNS configuration on my setup.

In my OpenIMS setup, I am running pcscf on one machine (say machine X) and
s/icscf and HSS on the other (machine Y). The DNS server is set on the same
machine (Y) running HSS and s/icscsf. The pcscf entry in the dnszone file
points to the machine X's address.

Now, when I try to register an X-lite user, it registers fine.

 

The same thing when I try with a Cisco phone, it does not work. The problem
lies with the phone's settings. I cannot assign an IP address for the proxy
address. To have the Cisco phone work properly, a DNS name must be assigned
to the proxy address. But since the pcscf is now on a different machine, the
proxy entry on the phone looks like this "pcscf.open-ims.test". This leads
to a DNS query for _sip._udp.pcscf.open-ims.test from the phone, which
doesn't look correct to me. The DNS server responds with a "no such name"
response. There is no such problem with X-Lite as X-Lite allows IP address
for proxy entry.

 

Can anyone give me some pointers to resolve this issue?

 

Thanks,

Rishin
