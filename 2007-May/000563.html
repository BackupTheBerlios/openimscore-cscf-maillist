Hi,

I have encountered same problem in fedora core5.

You should better check /opt/OpenIMSCore/ser_ims/modules/mysql/

If compilation is accomplished normaly, mysql.so is created in 
that directory.

When compiling OpenIMSCore, it uses mysql.h as include file.
So if mysql.so does not exist in that directory you should check 
that mysql.h is installed in your machine.
(Or check mysql.h is visible from compiler.)
