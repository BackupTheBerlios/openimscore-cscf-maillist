Hello Jose,

Having a default realm when none is found could be a good idea, although
it lacks flexibility.

Then, what about indicating and extracting the realm for TEL URLs from
the phone-context parameter?
[José] This could be done but there are situations where there is no phone-context ex. when the Tel URI contains an international number. Maybe the best thing would be extracting from the phone-context if this parameter exists and if it doesn't exist use the default realm.
