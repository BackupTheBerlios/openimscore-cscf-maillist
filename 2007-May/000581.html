> We were doing some tests with OpenIMSCore and the PSTN and found the need to add a few things to the source code. These changes consist on a adding a P-Asserted-Identity header like mentioned on 24.229 section 5.4.3.2 point b) for IMS initial requests (we exported a function to the script for this) and adding support for TEL URI's from PSTN to IMS (it was only supported sip URI because the function cscf_get_public_identity_from_requri only returned SIP URIs). With these changes we achieved to establish calls from and to PSTN with TEL URIs. We also added a few lines on the S-CSCF script to call the function that adds the P-Asserted-Identity header and to do an enum query (using ENUM module) to see if the TEL URI is an IMS user, as described on 24.229. 
>
> Since this might have some interest for the community we decided to post it but we would like you to take a look and see if you agree with what we done.
>
