Hi, 

 

I noticed that the S-CSCF script does not handle De-Registrations and Re-Registrations correctly, apparently everything is fine but when looking carefully you notice that S-CSCF does not send any Diameter message toward the HSS indicating that the user is no longer registered neither does it send a Diameter message indicating that there was a re-registration on that S-CSCF. 

The correct behaviour can be achieved by switching "S_update_contacts" function with the "S_assign_server" function.
