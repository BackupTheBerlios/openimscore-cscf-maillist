Hi Franz,

Well, that should not be a problem. It's the same port for everything,
yes. And SER core people will know that otherwise is a pain and just
very-very recent developments might make it easier. Anyway, the rules
need the following parameters:

- source IP
- source port
- destination IP
- destination port
- protocol.

So if 2 of those are always the same (IP:port of the P-CSCF) we still
have enough parameters left to identify the things correctly. And
anyway, if we would allocate unique P-CSCF ip:port / client then we
would run out of resources quite quickly on the P-CSCF.

But I think that you see some crash on the P-CSCF...

-Dragos
