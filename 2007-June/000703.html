We had some problems with our AS when parsing the P-Charging-Vector, this was because one of the P-Charging-Vector parameters is incorrect, the icid-generated-at parameter should not be quoted. 

 

I changed the following lines of the modules/pcscf/sip.c file:

 

static str p_charging_vector_s={"P-Charging-Vector: icid-value=\"",31};

- static str p_charging_vector_1={"\"; icid-generated-at=",22};

- static str p_charging_vector_2={"; orig-ioi=\"",13};

+static str p_charging_vector_1={"\"; icid-generated-at=",22};

+static str p_charging_vector_2={"; orig-ioi=\"",13};

static str p_charging_vector_e={"\"\r\n",3};
