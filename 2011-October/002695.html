Hi lb,

> Until this point everything looks ok. I checked clf.open-ims.test and port
3868
> they are opened when FHoSS is running otherwise the port gets closed. Is
it
> correct to assume that FHoSS provides CLF interface?

It provides the interface, but to setup thew connection you need the peer
side, the CLF.

> I see connection establishment from PCSCF to clf.open-ims.test:3868 and
> drops immediately. Do not why server is sending FIN to PCSCF.

Because the peer does not behave  as a CLF. The fact that you can send a
message to clf.open-ims.test:3868 does not mean that there is a CLF.
You need a CLF or anything which behaves like a CLF.
I recommend that you start with a trace of the messages exchanged and try to
analyze. 
I only can guess, and because I know that you need a CLF and you did not
provide a CLF I am sure that that is the reason.
Most probably some peer accepts connection setup but after capabilities
negotiation it turn out that the answering peer is not a CLF.

> Are you saying that I need to run emulator for e2 interface apart from
FHoSS? Could you please clarify?
E2 is the interface between P-CSCF and CLF. If you do not provide a real CLF
you need some software to simulate a CLF.

you want?
