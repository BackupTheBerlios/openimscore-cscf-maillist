Hi,
 
> What I meant with my UE processing path header is when my UE receives
> the 200 OK to it REGISTER and this 200 OK includes a path header (and service
> route header for that matter), then the UE will add these routes to any
> request the UE issues; it means e.g. when UE sends a SUBSCRIBE (to reg-
> event) the UE adds the routes it learned from REGISTER 200 OK (RFC 3327).

No, thats not the way it is defined.
The Service-Route header field (only) has to be used by the UE to route initial and standalone requests by using a proper Route header.
The Path header field is only for the S-CSCF which stores the values to route terminating requests properly. The UE has to ignore the Path header field.
