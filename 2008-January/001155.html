- IFC is configured with SESSION_CONTINUED for an AS that works in proxy
mode
- S-CSCF finishes triggering and forward
- next node responds with 5xx
- the ISC_orig/term_failure route incorrectly identifies the response
and tries to apply trigger check continuation.


Possible solution: the ISC_orig/term_failure route needs to be disabled
when there is a request that came back. However, this seems to involve
some state keeping if no clever tm ops are applied.
