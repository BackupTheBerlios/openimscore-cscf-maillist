Hi all,
 
I have a query on P-Charging-vector orig-ioi parameter related query
 
Section 5.4.3.3 
 
Point 7 says
 
When the S-CSCF receives, destined for a statically pre-configured PSI
or a registered served user, an initial request for a dialog or a
request for a standalone transaction, prior to forwarding the request,
the S-CSCF shall:
if there was no original dialog identifier present in the topmost Route
header of the incoming request store the value of the orig-ioi parameter
received in the P-Charging-Vector header, if present, and remove all 
received ioi parameters from the forwarded request if next hop is not an
AS;
 
which means when SCSCF will add orig-ioi parameter when the request is
forwarded to AS. So my query is when the request is forwarded to an AS,
will SCSCF add the stored orig-ioi parameter or add a new orig-ioi
parameter.
 
 
Point 4 says in the same section says about
 
insert a type 3 orig-ioi parameter in the P-Charging-Vector header. The
type 3 orig-ioi parameter identifies the sending network of the request
message before the received orig-ioi. The S-CSCF shall not include the
type 3 term-ioi parameter;
 
but this point doesn't make clear what it means "The type 3 orig-ioi
parameter identifies the sending network of the request message before
the received orig-ioi." Could any one explain this sentence from the
spec.
 
Thanks,
Sunil
