Hello Sunil,

a quick answer:
 
> 1.	Why is this header required (My assumption is cdr are needed to be
> generated)

There are several parameters with this header field with different tasks.
icid-value: for correlation of CDRs from different network nodes
orig-ioi and term-ioi: for inter-network charging

> 2.	How exactly the "icid" value is used

Roughly speaking: The first network node within the trusted area defines the
icid-value which is used for correlation of the distributed CDRs generated
by different network nodes.

> 3.	When session is released by SCSCF should'nt the BYE request have P-
> Charging-Vector header, if not why so ? 3gpp 24.229 does not mention of
> including it.

The P-Charging-Vector header is necessary for dialog-initiating and
standalone requests. Charging events within a dialog used the icid-value of
the P-Charging-Vector within the dialog-initiating request (e.g. INVITE).

Further details can be found in RFC 3455 and in the 3GPP standards of
32-series, e.g. 3GPP TS 32.260 for an overview.
