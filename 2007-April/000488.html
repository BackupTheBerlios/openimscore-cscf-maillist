Hi,

 

I am facing problem while trying to REGISTER a Cisco 7960 phone with FOKUS
IMS.

 

The ethereal trace shows that while sending back 401 - User Unauthorized for
the initial

REGISTER message, OSIMS sends the response message on the port which had
come as

Source Port of UDP header in the REGISTER message and *not* on what is
provided with 

the Contact header. However, the Cisco phone listens on 5060, which it had
provided with the

Contact header and misses the 401 response. As a result, Cisco phone keeps
on retransmitting

the REGISTER.

 

I tried to verify this behavior with X-Lite. The ethereal trace for X-Lite
shows that OSIMS is 

sending back 401 - User Unauthorized response to the Source Port of UDP
header which 

incidentally is the same one as provided with the Contact header. So,
Registration is working 

fine in the case of X-Lite.

 

Therefore, it appears to be a problem with FOKUS IMS.

 

Both the ethereal traces are attached. Could someone confirm my doubt?

 

Thanks,

Rishin
