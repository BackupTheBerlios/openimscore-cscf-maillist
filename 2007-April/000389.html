the resolv.conf should contain the IP address of the real DNS server.
However, this does not explain the crash.

To do a backtrace, you need first to find a core dump file (called
"core" or "core.12345" where 12345 was the pid of the crashed process).
This should be generated on a crash. If it is not, please do "ulimit -c
unlimited" before launching the icscf.sh.

Then run "gdb /opt/OpenIMSCore/ser_ims/ser <core_file>". There issue the
"bt" command and paste back the output.
