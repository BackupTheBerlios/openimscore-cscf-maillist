ser loads modules specified in  it configuration (the .cfg files - have a 
look).

in your case it seems to find the pcscf.o module, but not one of its 
dependencies.

make sure the LD_LIBRARY_PATH variable in the pcscf.sh file you use at 
startup points to the directory in which the lib_ser_cds.so resides.

or the other way around. if make sure lib_ser_cds.so is where 
LD_LIBRARY_PATH points to.
