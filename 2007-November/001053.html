The pcscf does not maintain the source port for sending terminating
requests and keepalives to clients. Instead it chooses the highest port
available.


This occurs when the pcscf is configured to listen to multiple ports.
To run multiple clients completely symmetrical from one host it is
necessery that the pcscf maintains the source and destination ports
symmetrical to provide NAT traversal in all cases.


Example:
PCSCF listens to port 5060 and 5061
Client registers from port 5060 to port 5060
PCSCF pings keepalive from port 5061 to 5060 (which does not work for a
symmetric restricted NAT)
PCSCF transmits UE terminating requests from port 5061 to port 5060 as
well.
