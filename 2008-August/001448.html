Hi,

I've found the 08_cdp_sessions branch P-CSCF has a bug that parses the 
qop parameter incorrectly during registration.
The WWW-Authenticate header in the 401 Challenge response looks as follows:

WWW-Authenticate: Digest realm="qos-ims.test", 
nonce="g6nhN4EMyhM+ZUrFws/XyIt1taLv2wAA41yPbD5CLws=", 
algorithm=AKAv1-MD5qop="auth,auth-int"

This causes a client crash - the problem is not serious as disabling the 
qos parameter in the S-CSCF config file solves it.

I have attached a Wireshark trace of the registration.
