Hi Julio,

> > Does it mean that if I add two sub-domains of my domain;
> > mrfp.domain.com and mrpf1.domain.com in the mysql table, the OpenIMS
> > core will accept messages coming from this 2 sub-domains? Or should I
> > use PSI for each of two different boxes?

As I understand there are two different questions/answers:

1. Network domain security:
You do not need to enable an IP-address of a server somewhere in OpenIMSCore
as long as it is part of the domain. So any request with a target within
"domain.com" should be handled properly within the domain.

2. Routing requests to the new box:
This depends on the role of the box. In case of a media server it is
probably the best solution to attach the servers to the platform via PSIs.
Than it does not need to register and requests addressed to it are properly
routed within the platform.
