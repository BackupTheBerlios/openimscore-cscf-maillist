Hi Julio,
 
> I am looking for instructions how to connect a 3-party network element
> to the OpenIMS core (I don't mean to say Application Server).
> 
> I have experience with commercial system. They have a page (or menu if
> you will) in the s-cscf named "authorized elements"  in which you
> defined all the elements that are allowed to connect to the core like
> the i-cscsf, etc.

This system obviously has implemented the network domain security in that
stringent way.

> Is there such a thing in the OpenIMS core? If not, how you connect
> such external "core" elements like a Media Server?

In OpenIMSCore there is also network domain security implemented. It is
based on domain names. There is a list of trusted domains handled by the
I-CSCF (in database table of icscf "nds_trusted_domains"). If a request is
received from a domain not on that list (based on a check of the Via header
field) all trust relevant header fields are stripped before further
processing. So if the server you want to connect uses an address of the home
domain then it is automatically trusted.

The "how to connect" question maybe requires more details on the role of the
server and how it will be addressed. You might perhaps use PSIs (Public
Service Identities).
