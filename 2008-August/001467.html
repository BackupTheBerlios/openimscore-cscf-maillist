I have attached the scenario and can explain why the MESSAGE request iis
rejected by P-CSCF.

The P-CSCF cares for network security and rejects requests (other than
REGISTER) coming from not registered users. The criteria for this check are
the parameters "protocol, host and port" of the Via header field, which must
match with existing registered users in P-CSCF.

The REGISTER request used "UDP 192.168.3.9:5061".
The MESSAGE request used "UDP 192.168.3.7:6060".

Therfore the MESSAGE request is rejected with "403 Forbidden - Not
Registered! You must register first with a S-CSCF".
