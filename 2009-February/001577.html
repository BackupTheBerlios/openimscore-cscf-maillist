> "400 Bad Request - Not following indicated service routes"
> 
> The REFER is going to P-cscf and terminationg there....it is not reaching
> to s-cscf also!

It is clear that the request is not forwarded by P-CSCF to S-CSCF because of
the above issue. It is one of the fundamental tasks of a P-CSCF to check if
terminals honour the Service-Route indicated during registration. 

> I also see there is no handling for REFER in P-cscf.cfg and S-
> cscf.cfg.should i write code for handling REFER Method or openims already
> handles this REFER Method.....

No. This is not necessary. REFER requests should be handled perfectly by the
existing algorithms of pcsdcf.cfg.
You have to solve your service-Route problem first. Then it will most
probably work fine. 
If you need some hints in this issue please add a trace of the signalling
messages.
