The INVITE request (packet 4527) contains a Route header (derived from 200
OK response of REGISTER request) but the REFER request (packet 5751) does
not. That's the problem.

P-CSCF may as an option correct this insufficiency in pcscf.cfg. Look at the
code and use Variant 2:

	if (!P_follows_service_routes()){		
		#Variant 1 - deny access to the network
		sl_send_reply("400","Bad Request - Not following indicated
service routes");
		break;
		#Variant 2 - enforce routes and let the dialog continue
   		#P_enforce_service_routes();
	}
