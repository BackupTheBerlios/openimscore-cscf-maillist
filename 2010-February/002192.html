Hi
I would like to use different address spaces in my access network and 
the core network. My P-CSCF is configured to listen to 10.1.0.10 
(access) as well as 172.28.0.10 (core). What happens when I send a 
request from a client on the access network to the access address of the 
P-CSCF it becomes forwarded to the core network also using the access 
address, e.g.

Register from 10.1.0.99 ---> 10.1.0.10 becomes forwarded as
Register from 10.1.0.10 ---> 172.28.0.10

I would like to see that the P-CSCF used the core address when 
forwarding towards the core network. Is it possible make that happen 
through the configuration file?

The same thing happens in the opposite direction, when forwarding the 
request the P-CSCF uses the source address of the core network instead 
of the access address.
