Hi all,

I have in my lab an OpenIMScore in order to build a test bed. Such IMS core is well installed and is running. I have the Rx interface activated to an SPDF that is under development.
However, I have observed something curious when I try to make a call between two users that are registered in the same domain (in my case: open-ims.test).
These two users are "bob" (already inserted when installing FHoSS) and "lena" (added by me). Both users registers in the S-CSCF but when I try to make a voice call from "bob" to "lena" and when I capture the Diameter messages exchanged in Rx interface I always see the following:

AAR message regarding "lena" user: Authorization-Lifetime AVP value is always set to 60 seconds
AAR message regarding "bob" user: Authorization-Lifetime AVP value is always set to 300 seconds.

Therefore whenever I try to maintain the voice call longer than 60 seconds the P-CSCF sents a STR message with cause: DIAMETER_ADMINISTRATIVE for the session ID related to "lena" and the voice call automatically hangs up.

There is no way to make the PCSCF insert a greater value and I don't know why two users of the same OpenIMScore have two different Authorization-Lifetime values.

At registration time, both users have 3600 seconds in the AARs generation by REGISTER SIP message.

Does anybody know if this issue is a bug and how to solve it?

Thank you very much.

Victor
